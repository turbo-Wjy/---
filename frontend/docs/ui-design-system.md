# AI职教智学工坊 UI 设计规范

## 目标

第一阶段以首页工作台为样板页，建立可复用的后台工作台视觉系统。所有页面优先使用 Ant Design 组件和项目封装组件，禁止在业务页面中随意复制卡片、标题、进度条、标签等样式。首页采用“轻量概览 + 专项详情页”结构，默认页只承载关键摘要和入口，完整列表进入二级页面处理。

## 视觉方向

- 产品类型：AI 职教学习平台、后台工作台、学习成长仪表盘。
- 关键词：清晰、可信、智能、轻量、可持续扩展。
- 主要视觉特征：深色侧边栏、浅灰蓝页面背景、AntD 默认白色信息卡片、蓝紫主视觉、少量绿色/橙色表示进度和状态。

| `--sc-color-primary-soft` | 选中背景、弱强调 | `#ecf2ff` |
| `--sc-color-sidebar` | 侧边栏背景 | `#06184a` |

### 字体

- 字体族：系统 UI 字体，优先 `Inter`、`PingFang SC`、`Microsoft YaHei`。
- 页面标题：28px / 700。
- 卡片标题：16px / 700。
- 模块标题：16px / 700。
- 正文：14px / 400。
- 辅助文字：12px / 400。
- 数字指标：28px / 700，使用等宽数字特性。

### 间距

- 基础网格：4px。
- 页面边距：24px。
- 卡片间距：16px。
- 卡片内边距：20px 或 24px。
- 列表项间距：12px。
- 图标和文字间距：8px。

### 圆角和阴影

- 导航选中项：8px。
- 普通卡片：10px。
- Hero Banner：12px。
- 按钮：999px 用于胶囊主按钮，普通按钮沿用 AntD 8px。
- 卡片阴影：业务卡片优先使用 AntD 默认边框和轻阴影，避免自定义强发光效果。
- 浮层阴影：使用 AntD 默认浮层阴影。

## 布局规则

- PC 优先，当前最小宽度 1200px。
- 左侧导航固定 248px。
- 顶部栏高度 72px。
- 页面内容必须铺满可用内容区宽度，不使用固定居中的窄版容器；每个页面都按当前视口自适应。
- 首页主内容为左右两栏：主区域自适应，右侧栏 340px 到 360px。
- 首页默认页不放页内 Tabs，不显示大标题“首页工作台”，直接进入概览内容。
- 顶部必须显示“一级菜单 / 当前页面”面包屑，例如“首页工作台 / 今日概览”。
- 首页只展示 Hero、4 个指标、今日优先处理、学习路径进度、学习提醒预览、推荐资源预览、AI 助手入口。
- 卡片和列表优先保持信息密度，避免营销式大留白，也避免把完整业务列表都堆在首页。
- 页面里的按钮必须具备下钻或跳转行为，不能只做静态展示；业务未完成时也要跳转到对应占位页或上级概览页。
- 页面必须调用真实接口；接口成功有数据时展示真实数据。
- 接口成功但数据不足、或接口失败时，允许使用 `src/mocks/**` 的展示数据补齐视觉效果，但不得显示 `Network Error` 等底层错误文本。
- 新增、提交、审核、删除、确认等会改变业务状态的操作必须走真实接口，不能用 mock 伪装成功。
- 关系型知识图谱统一使用 G6，统计图表统一使用 G2；业务页面不得手写 SVG 或按钮云伪装图谱。
- 复杂页面第一屏只放核心图谱或核心图表，明细表格、资源入口和过程记录进入下方 Tabs。

## 组件使用规则

- 卡片必须使用 `AppCard` 或 AntD `Card variant="outlined"`。
- 区块标题必须使用 `SectionHeader`。
- 首页指标必须使用 `MetricCard`，数据从 `DashboardOverview.metrics` 映射。
- 状态标签必须使用 `StatusTag`。
- 进度条必须使用 `ProgressBar`。
- 顶栏、侧栏由 `AppTopbar`、`AppSidebar` 统一维护。
- 课程、融合图谱、短板定位和成长路径等可视化页面必须复用 `G2ChartShell`、`G6GraphShell` 或其业务封装组件。

## 还原验收清单

- 侧边栏深色渐变、选中态、AI 助手入口还原。
- 顶部搜索框、通知、消息、头像区域还原。
- Hero Banner 蓝紫主视觉、机器人视觉和主按钮保留，但高度和装饰复杂度要克制。
- 指标卡片四列，使用默认白卡、轻量图标和进度条。
- 首页展示今日优先处理、学习路径进度、学习提醒预览、推荐资源预览、AI助手入口。
- 学习提醒最多 3 条，推荐资源最多 2 条，完整内容进入二级页面。
- 文本不溢出，按钮和图标具备可访问名称。
