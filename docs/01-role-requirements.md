# AI 岗课赛证学习平台角色需求说明

## 1. 角色设计目标

系统角色设计服务于以下主线：

对话式学习画像构建 → 多智能体资源生成 → 学习路径规划与资源推送 → 智能辅导 → 学习效果评估 → 岗课赛证成果沉淀 → AI 简历生成 → 学生画像与统计导出。

角色设计采用“登录角色 + 职责标签 + 模块权限 + 数据范围”的方式，避免把任课老师、小组负责教师、带队老师等相近身份拆得过细。

## 2. 登录角色清单

| 序号 | 角色 | 定位 | 是否核心 |
| --- | --- | --- | --- |
| 1 | 系统管理员 | 账号、权限、菜单、AI 智能体配置、系统日志 | 是 |
| 2 | 专业负责人 | 学生账号导入、岗位审核、证书标准导入、画像和分类统计导出 | 是 |
| 3 | 教师 | 统一承载任课老师、小组负责教师、带队老师职责 | 是 |
| 4 | 竞赛管理员 | 发布竞赛、审核竞赛成果和荣誉展示 | 是 |
| 5 | 企业导师 | 发布岗位、审核简历、提交或推荐简历 | 是 |
| 6 | 学生 | 构建画像、生成资源、学习答题、上传成果、生成简历、岗位投递 | 是 |
| 7 | 数据查看者 | 只读查看统计、画像分析和学习效果 | 可选 |

## 3. 权限设计原则

### 3.1 登录角色

登录角色决定用户能进入哪些端和一级功能，例如学生端、教师端、企业导师端、后台管理端。

### 3.2 职责标签

教师角色内部通过职责标签区分实际工作内容：

| 职责标签 | 适用对象 | 主要权限 |
| --- | --- | --- |
| 任课老师 | 课程教师 | 上传课程资料、维护课程知识点 |
| 小组负责教师 | 分组指导教师 | 审核学生简历、查看负责学生画像 |
| 带队老师 | 竞赛指导教师 | 上传竞赛成果、维护参赛材料 |

同一名教师可以拥有多个职责标签，不需要创建多个账号。

### 3.3 模块权限

模块权限控制用户在具体模块中的操作能力，例如查看、新增、编辑、导入、导出、审核、发布、生成、推荐等。

### 3.4 数据范围

| 数据范围 | 说明 | 适用角色 |
| --- | --- | --- |
| 全平台 | 可管理系统配置和全部基础数据 | 系统管理员 |
| 专业 | 仅可管理绑定专业的数据 | 专业负责人 |
| 课程 / 分组 / 带队范围 | 仅可查看和管理自己负责范围内的数据 | 教师 |
| 竞赛范围 | 可管理竞赛发布、成果审核和荣誉展示 | 竞赛管理员 |
| 企业 / 岗位 | 仅可查看自己企业或自己发布岗位相关数据 | 企业导师 |
| 个人 | 仅可查看和维护个人数据 | 学生 |
| 只读范围 | 只读查看授权范围内统计 | 数据查看者 |

## 4. 角色需求

### 4.1 系统管理员

定位：平台级系统管理角色，不直接参与业务审核。

主要权限：

- 管理账号、角色、权限、菜单。
- 配置 AI 智能体、模型参数、资源生成类型。
- 配置学院、专业、班级、课程、企业等基础数据。
- 配置初始密码、首次登录改密、账号启停规则。
- 查看系统日志、操作日志、模型调用日志。
- 管理数据字典、系统参数、菜单配置。

限制：

- 默认不参与岗位、简历、竞赛成果、证书成果等业务审核。

### 4.2 专业负责人

定位：专业范围内的数据和审核负责人。

主要权限：

- 导入本专业学生 Excel，生成学生账号。
- 审核企业导师发布的岗位，审核通过后岗位进入职位广场。
- 导入和维护本专业证书标准、考证资料、证书题库。
- 查看本专业学生学习画像、学习效果、岗位能力、竞赛成果、证书达标情况。
- 导出本专业学生画像表格和分类统计表格。
- 查看本专业岗课赛证达成统计。

限制：

- 只能管理绑定专业的数据。
- 不发布企业岗位。
- 不代替学生确认 AI 简历或投递岗位。

### 4.3 教师

定位：统一教师角色，通过职责标签承接任课老师、小组负责教师、带队老师职责。

主要权限：

- 任课老师：上传课程资料，维护课程知识点，查看课程学习记录。
- 小组负责教师：查看负责学生画像，审核学生简历，可打回学生修改。
- 带队老师：上传竞赛成果和荣誉材料，维护参赛材料。

限制：

- 只能查看课程、分组或带队范围内的数据。
- 简历审核后可通过或打回，但不能代替学生确认投递。

### 4.4 竞赛管理员

定位：竞赛信息和荣誉展示管理角色。

主要权限：

- 发布竞赛名称、时间、地点、参赛要求、官方链接、报名方式等信息。
- 审核带队老师上传的竞赛成果和荣誉材料。
- 审核通过后发布到荣誉展示和融合图谱。
- 查看竞赛参与与获奖统计。

限制：

- 不审核岗位和企业简历。
- 不修改学生画像生成结果。

### 4.5 企业导师

定位：企业侧岗位与简历处理角色。

主要权限：

- 发布岗位信息。
- 修改被打回的岗位并重新提交审核。
- 查看自己发布岗位的审核状态。
- 审核学生投递且已经通过小组负责教师审核的简历。
- 可打回学生修改简历。
- 审核通过后提交或推荐给企业。
- 反馈面试、待沟通、淘汰、录用等状态。

限制：

- 不能直接查看全体学生。
- 只能查看流转到企业审核节点的简历。
- 不能修改学生画像、课程记录、竞赛记录、证书记录。

### 4.6 学生

定位：系统主要使用者和数据主体。

主要权限：

- 使用学号登录，首次登录后强制修改初始密码。
- 通过自然语言对话构建学习画像。
- 调用多智能体生成课程讲解文档、PPT、题库、思维导图、拓展阅读、视频 / 动画脚本、代码案例、实践项目材料。
- 查看学习路径、资源推送、智能辅导和学习效果评估。
- 浏览课程资料，在线学习，答题练习，形成学习记录。
- 查看职位广场，基于画像和岗课赛证成果生成 AI 简历。
- 编辑确认 AI 简历后提交岗位投递。
- 查看竞赛信息、证书标准、项目实训内容。
- 上传个人证书成果等材料。
- 查看个人画像、投递记录、学习记录和审核进度。

限制：

- AI 简历必须由学生确认后才能提交审核。
- 不能查看其他学生隐私数据。
- 学号、姓名、专业、班级等教务基础字段默认不可自行修改。

### 4.7 数据查看者

定位：只读统计查看角色，可用于学院领导或管理者。

主要权限：

- 查看学生画像统计。
- 查看学习效果统计。
- 查看岗课赛证达成统计。
- 查看专业、班级、年级等分类分析。

限制：

- 默认只读。
- 不参与账号管理、资源发布、审核流转和数据修改。

## 5. AI 智能体说明

AI 智能体是系统内部服务能力，不作为登录用户。

| 智能体 | 核心能力 |
| --- | --- |
| 画像构建智能体 | 对话抽取学生画像特征 |
| 资源设计智能体 | 拆解学习需求并调度资源生成 |
| 文档生成智能体 | 生成讲解文档和学习资料 |
| PPT 生成智能体 | 生成 PPT 学习材料 |
| 题库生成智能体 | 生成练习题、测验题和错题强化题 |
| 思维导图智能体 | 生成知识结构和学习脉络 |
| 视频 / 动画脚本智能体 | 生成讲解脚本、分镜和动画说明 |
| 实操案例智能体 | 生成代码案例和实践项目材料 |
| 学习路径规划智能体 | 规划学习目标、步骤和顺序 |
| 资源推荐智能体 | 按画像推送资源 |
| 智能辅导智能体 | 提供即时答疑和多模态解释 |
| 学习评估智能体 | 分析学习效果并调整策略 |
| 简历生成智能体 | 根据画像和成果生成简历初稿 |
| 统计分析智能体 | 生成画像统计和分类统计 |

## 7. 专业负责人导出内容

专业负责人支持导出：

- 学生基础信息表。
- 学生画像汇总表。
- 学习画像维度统计。
- 岗位能力分类统计。
- 课程学习进度统计。
- 学习效果评估统计。
- 竞赛参与与获奖统计。
- 证书达标统计。
- 项目实训完成统计。
- 简历完善度与投递状态统计。
